# ターミナル基本操作

TypeScriptを学ぶ上で、ターミナル（コマンドライン）の基本操作を身につけることは必須です。ターミナルを使えるようになると、開発の幅が大きく広がります。

**前提知識：** このページを読む前に、[ファイルとフォルダの基礎](file_directory)を先に読むことをおすすめします。ホームディレクトリやカレントディレクトリなどの概念を理解していると、このページの内容がずっと分かりやすくなります。

## ターミナルとは？

ターミナル（またはコマンドライン、シェル）は、文字だけでコンピュータを操作するためのツールです。マウスでアイコンをクリックする代わりに、キーボードでコマンド（命令）を入力して操作します。

## 使用するターミナル

### Macユーザーの方

Macに標準でインストールされている**ターミナル**を使用します。

- Finderで「アプリケーション」→「ユーティリティ」→「ターミナル」を開く
- またはSpotlight検索（⌘ + Space）で「ターミナル」と入力して開く

### Windowsユーザーの方

環境構築でインストールした**Ubuntu（WSL）**を使用します。

- VS Code上でUbuntuのターミナルを開く（[VS CodeでWSLのターミナルを開く方法](https://qiita.com/_masa_u/items/d3c1fa7898b0783bc3ed)を参照）
- またはスタートメニューから「Ubuntu」を検索して開く

**重要：** WindowsのコマンドプロンプトやPowerShellではなく、必ずUbuntu（WSL）を使用してください。Ubuntu/WSLは**Linux環境**なので、Macと同じコマンドが使えます。

## 基本的なコマンド

ターミナルで使用する基本的なコマンドを覚えましょう。

### pwd - 現在のディレクトリを表示

```bash
pwd
```

**pwd**は「**P**rint **W**orking **D**irectory」の略で、現在どのディレクトリにいるか（カレントディレクトリ）を表示します。

例：
```bash
$ pwd
/Users/tanaka/Documents
```

これは「今、`Users`フォルダの中にある`tanaka`フォルダの中にある`Documents`フォルダにいますよ」という意味です。

`/Users/tanaka/Documents`というパスの読み方：
- `/` = ルート（一番上の階層）
- `Users` = ルートの中にあるUsersフォルダ
- `tanaka` = Usersの中にあるtanakaフォルダ（あなたのホームディレクトリ）
- `Documents` = tanakaの中にあるDocumentsフォルダ（今ここにいる）

### ls - ディレクトリ内のファイル一覧を表示

```bash
ls
```

**ls**は「**L**i**s**t」の略で、現在のディレクトリにあるファイルやフォルダの一覧を表示します。

例：
```bash
$ ls
Desktop  Documents  Downloads  Pictures
```

### cd - ディレクトリを移動

**cd**は「**C**hange **D**irectory」の略で、ディレクトリを移動するコマンドです。

#### 特定のフォルダに移動

```bash
cd フォルダ名
```

例えば、`cd Documents`でDocumentsフォルダに移動します。

#### 親ディレクトリ（一つ上）に移動

```bash
cd ..
```

`..`は親ディレクトリを意味します。一つ上の階層に戻ります。

例：
```bash
$ pwd
/Users/tanaka/Documents/myproject
$ cd ..
$ pwd
/Users/tanaka/Documents
```

この例では：
1. 最初は`/Users/tanaka/Documents/myproject`にいた
2. `cd ..`で一つ上の階層（親ディレクトリ）に移動
3. 移動後は`/Users/tanaka/Documents`にいる
4. つまり、`myproject`フォルダから`Documents`フォルダに移動したということ

#### ホームディレクトリに移動

```bash
cd ~
```

`~`（チルダ）はホームディレクトリを意味します。どこにいても、このコマンドで自分のホームディレクトリに戻れます。

#### ルートディレクトリに移動

```bash
cd /
```

`/`はルートディレクトリ（一番上の階層）を意味します。

### ディレクトリを作成

```bash
mkdir フォルダ名
```

新しいフォルダを作成します。例えば、`mkdir myproject`で「myproject」というフォルダが作られます。

### ファイルやディレクトリを削除

```bash
rm ファイル名
```

ファイルを削除します。

```bash
rm -r ディレクトリ名
```

ディレクトリとその中身を全て削除します。（**注意：** 取り消しできません）

## 実際に試してみよう

ターミナルを開いて、以下の手順で練習してみましょう。

```bash
# 1. 現在地を確認
pwd

# 2. ホームディレクトリに移動
cd ~

# 3. 練習用フォルダを作成
mkdir terminal_practice

# 4. 作成したフォルダに移動
cd terminal_practice

# 5. 現在地を確認
pwd

# 6. 一つ上のディレクトリに戻る
cd ..

# 7. 練習用フォルダを削除
rm -r terminal_practice
```

## 便利なショートカット

開発を効率的に進めるために、最低限のショートカットを覚えておきましょう。

### VS Codeでのショートカット

| 操作 | Windows | Mac |
|------|---------|-----|
| **コピー** | Ctrl + C | ⌘ + C |
| **ペースト** | Ctrl + V | ⌘ + V |
| **全選択** | Ctrl + A | ⌘ + A |
| **行を選択** | Ctrl + L | ⌘ + L |

### ターミナルでのショートカット

| 操作 | Windows（Ubuntu） | Mac |
|------|------------------|-----|
| **コピー** | Ctrl + Shift + C | ⌘ + C |
| **ペースト** | Ctrl + Shift + V | ⌘ + V |
| **実行中のコマンドをキャンセル** | Ctrl + C | Ctrl + C |
| **Tab補完** | Tab | Tab |
| **履歴を呼び出す** | ↑（上矢印） | ↑（上矢印） |

### ショートカットについての注意点

**VS Codeのショートカットを全てマスターする必要はありません。** ここで挙げている最低限のもの（コピー、ペースト、全選択、行選択）を知っていれば十分です。

それ以外のショートカットは、必要に応じて少しずつ覚えていけば大丈夫です。そこまで気にせず、まずは基本的な操作に慣れることを優先しましょう。

## ターミナルを使う上でのポイント

- **Tab キーで補完**
  コマンドやファイル名を途中まで入力してTabキーを押すと、自動で補完されます。とても便利なので積極的に使いましょう。

- **上矢印キーで履歴を呼び出す**
  以前に入力したコマンドを上矢印キーで呼び出せます。同じコマンドを何度も入力する手間が省けます。

- **Ctrl + C でキャンセル**
  ターミナルで実行中のコマンドを途中でキャンセルしたいときは、**Ctrl + C**を押します（MacもWindowsも同じ）。間違えてコマンドを実行してしまったときなどに使えます。

- **エラーを恐れない**
  間違ったコマンドを入力してもコンピュータが壊れることはありません。エラーメッセージをよく読んで、何が問題なのか理解しましょう。
