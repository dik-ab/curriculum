# ファイルとフォルダの基礎

ターミナルを使う前に、コンピュータの中でファイルやフォルダがどのように整理されているのかを理解しましょう。この基礎知識があると、ターミナルの操作がずっと分かりやすくなります。

## ファイルとは？

**ファイル**は、データを保存する単位です。普段使っているもので例えると：

- **テキストファイル** - メモ帳で書いた文章（`.txt`）
- **画像ファイル** - 写真やイラスト（`.jpg`, `.png`）
- **プログラムファイル** - JavaScriptやHTMLのコード（`.js`, `.html`）
- **ドキュメント** - Word文書やExcelファイル（`.docx`, `.xlsx`）

ファイルには必ず**名前**があり、多くの場合**拡張子**（`.txt`、`.jpg`など）がついています。拡張子は、そのファイルがどんな種類のデータなのかを示しています。

## フォルダ（ディレクトリ）とは？

**フォルダ**（または**ディレクトリ**）は、ファイルをまとめて整理するための入れ物です。

現実世界で例えると：
- **本棚** = コンピュータ全体
- **棚の段** = フォルダ
- **本** = ファイル

フォルダの中には、ファイルだけでなく、他のフォルダを入れることもできます。これを**入れ子構造**と言います。

## ディレクトリ構造

コンピュータの中では、フォルダとファイルが階層的に整理されています。これを**ディレクトリ構造**または**ツリー構造**と呼びます。

### 例：家のディレクトリ構造

```
家（ルート）
├── リビング
│   ├── テレビ
│   └── ソファ
├── キッチン
│   ├── 冷蔵庫
│   └── 調理器具
│       ├── 包丁
│       └── フライパン
└── 寝室
    ├── ベッド
    └── クローゼット
```

### コンピュータのディレクトリ構造

```
ルート（/）
├── Users（ユーザーフォルダ）
│   └── あなたの名前（ホームディレクトリ）
│       ├── Documents（ドキュメント）
│       │   ├── report.txt
│       │   └── project
│       │       └── index.html
│       ├── Downloads（ダウンロード）
│       └── Desktop（デスクトップ）
└── Applications（アプリケーション）
```

## 重要な概念

### 1. ホームディレクトリ

**ホームディレクトリ**は、あなた専用のフォルダです。記号では`~`（チルダ）で表されます。

- **Mac**: `/Users/あなたの名前/`
- **Windows（Ubuntu）**: `/home/あなたの名前/`

デスクトップ、ドキュメント、ダウンロードなどのフォルダは、すべてこのホームディレクトリの中にあります。

### 2. カレントディレクトリ（現在のディレクトリ）

**カレントディレクトリ**は、今自分がいる場所のことです。「今いるフォルダ」と考えると分かりやすいでしょう。

例えば、エクスプローラー（Windows）やFinder（Mac）でフォルダを開いている時、そのフォルダがカレントディレクトリです。

ターミナルでも同じように、常に「どこかのディレクトリ」にいる状態で作業します。

### 3. 親ディレクトリ

**親ディレクトリ**は、カレントディレクトリの一つ上の階層のディレクトリです。記号では`..`（ドット2つ）で表されます。

例：
```
/Users/あなたの名前/Documents/project
```
という場所にいる時、親ディレクトリは：
```
/Users/あなたの名前/Documents
```
です。

### 4. ルートディレクトリ

**ルートディレクトリ**は、すべてのディレクトリの一番上（根っこ）にあるディレクトリです。記号では`/`で表されます。

コンピュータの中の全てのファイルとフォルダは、このルートディレクトリの中に存在しています。

## パス（Path）とは？

**パス**は、ファイルやディレクトリの場所を示す住所のようなものです。

### 絶対パス

ルートディレクトリから始まる完全な住所です。

例：
```
/Users/tanaka/Documents/myproject/index.html
```

### 相対パス

カレントディレクトリからの相対的な場所を示します。

例（カレントディレクトリが`/Users/tanaka/Documents`の時）：
```
myproject/index.html
```

## 図で理解する

```
/（ルート）
│
└── Users
    └── tanaka（ホームディレクトリ: ~）
        ├── Documents ← 今ここにいる（カレントディレクトリ）
        │   ├── report.txt
        │   └── myproject
        │       └── index.html
        ├── Downloads
        └── Desktop
```

この状態で：
- **ホームディレクトリ**: `/Users/tanaka`または`~`
- **カレントディレクトリ**: `/Users/tanaka/Documents`または`~/Documents`
- **親ディレクトリ**: `/Users/tanaka`（一つ上に戻る = `..`）
- **ルートディレクトリ**: `/`（一番上）

## なぜこれを理解する必要があるのか？

ターミナルでは、マウスでフォルダをクリックする代わりに、文字でこれらの場所を指定します。

例えば：
- `cd ~/Documents` → ホームディレクトリのDocumentsに移動
- `cd ..` → 親ディレクトリ（一つ上）に移動
- `cd /` → ルートディレクトリに移動

この基礎知識があれば、ターミナルのコマンドが何をしているのかがすぐに理解できるようになります。
